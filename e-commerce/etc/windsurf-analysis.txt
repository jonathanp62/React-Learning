# Potential Issues / Inconsistencies

- **Unused dependency**: `react-toastify` is installed but not used (using `react-hot-toast` instead).

- **Performance**:
  - Search dropdown filters `products` on every change; add debounce for better UX on slower machines.
  - Repeated `description.split(" ").slice()` on every render; consider precompute or memoization if needed.
- **Resilience**:
  - No retry/backoff on product fetch.
  - No loading skeletons; simple spinner only.
- **Testing**:

# Recommendations

- **Unify filtering**:
  - Move filter computation into Redux using selectors (e.g., `createSelector` with inputs of `data`, `selectedCategory`, `selectedPrice`, `searchTerm`) and derive `filtered` there rather than storing it.
  - Add `searchTerm` to Redux; let `Navbar` dispatch updates and `Home` simply read the derived `filtered`.
- **Improve Search UX**:
  - Debounce input (300ms).
  - Close dropdown on outside click and on route change.
  - Keyboard navigation and ARIA roles for listbox/options.
- **Error and Toasting**:
  - Replace `alert` with `toast.error`.
  - Add error boundary for the app or for `Home`.
- **Cart Enhancements**:
  - Persist `cart` to `localStorage`.
  - Add quantity management instead of pushing duplicates; shape could be `{id, qty, product}`.
  - Compute totals via memoized selector.
- **Code Quality**:
  - Extract constants for price ranges.
  - Add types or migrate to TypeScript gradually (start with slices and components props).
  - Add unit tests for reducers and selectors.
- **UI/Accessibility**:
  - Add labels for search.
  - Add focus management when dropdown opens.
  - Empty states with guidance; skeleton loaders for products.
- **Cleanup**:
  - Remove `react-toastify` if unused or migrate all to it.
  - Update README to match actual implementation (fetch vs axios).
- **Deployment**:
  - CRA is fine for learning; for production consider Vite + code splitting + environment variables.
